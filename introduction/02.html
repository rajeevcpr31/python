All You Need to Know About Strings

In Python, strings are one of the most widely used data types. When you create a string, each character in that string is stored as Unicode in memory. Unicode is a universal character encoding system that assigns a unique code point to every character across all languages. For example, A has the Unicode value 65, while the emoji ðŸ˜€ has the Unicode value 128512. You can check the Unicode of any character using the ord() function and convert it back to a character using the chr() function.

print(chr(65)) # A

Strings in Python support indexing, which allows you to access individual characters directly. Indexing starts at 0 and goes up to the length of the string minus one. Negative indexing is also supported, which lets you access characters from the end. For example, if greet = "Hello World", then greet[0] will give H and greet[-1] will give d.

Slicing is another powerful feature of strings. It allows you to extract a portion of the string by providing a start index, an end index (exclusive), and an optional step value. If the start index is not given, it begins from the start of the string. If the end index is not given, it goes till the end. If the step is not provided, it defaults to 1. For instance, greet[0:5] results in Hello, greet[0::2] gives HloWrd, and greet[:5] also results in Hello. However, it's important to remember that strings are immutable, so you cannot change a slice directly. For example, trying greet[0:2] = "Hi" will raise a TypeError saying that 'str' object does not support item assignment.

A very important property of strings is that they are immutable. Once created, the contents of a string cannot be changed. Any modification, like concatenation or replacement, creates a new string object instead of altering the original. For example, "Hello" + "World" creates a new string HelloWorld, and "Hi" * 3 creates HiHiHi. This immutability makes strings safe to use in many situations but is also the reason why operations that heavily modify strings can be slower compared to mutable types.

Strings also support membership operations using in and not in. For example, "a" in "cat" returns True, while "dog" not in "cat" also returns True. For storing longer blocks of text, Python supports multiline strings, which can be created using triple quotes (""" or '''). If you want to treat backslashes literally instead of as escape sequences, you can use raw strings by prefixing the string with r, for example r"C:\path\to\file".

Sometimes, you may need to work with encoded text, especially when dealing with files or networks. Strings can be converted to bytes using the .encode() method and converted back using .decode(). This is particularly important when working with different character encodings like UTF-8.

Finally, Python makes string formatting easy. You can insert variables into strings using f-strings, which are fast and clean: f"My name is {name}". Other options include the older .format() method and % formatting, but f-strings are the modern and preferred way.

There are also some advanced points about strings that are commonly asked in interviews. One of them is performance in string concatenation. Since strings are immutable, repeatedly using + inside a loop creates many temporary string objects, which is inefficient. Instead, it's better to collect all parts into a list and then use "".join(list) to build the final string efficiently.

Another interesting concept is string interning. Python sometimes stores only one copy of certain strings (especially small or frequently used ones) to save memory. This is why you might see cases where two identical strings point to the same object in memory. For example, "hello" is "hello" may return True because of interning, but this behavior should not be relied upon for program logic — always use == for value comparison.

Getting Input from Users

In Python, user input is taken using the input() function. This function always returns the entered value as a string. It can be called either without any argument or with a prompt string that appears on the console. In both cases, the user's input is returned as a string. Since the return type is always a string, you can use explicit type conversion (like int() or float()) to convert it into another data type when needed.

x = input()
print(x)  # x is a string

age = int(input("Enter your age: "))
print(f"Your age is {age}")  # age is now an integer

If you want to take multiple values in a single line, you can combine input() with the .split() method. When a user enters multiple values separated by spaces, input() reads the entire line as one string. The .split() method then divides it into smaller parts and returns a list of substrings. You can assign these parts to variables directly:

n1, n2, n3 = input("Enter 3 numbers separated by space: ").split()
print(f"{n1} + {n2} + {n3} = {n1 + n2 + n3}")  
# n1, n2, n3 are strings, so concatenation happens

In this example, although the user enters numbers, Python still treats them as strings. So, the + operator performs string concatenation instead of addition. To fix this, we can use the map() function, which applies a given function to each element of an iterable and returns an iterator with the results.

For instance, map(int, "5") applies the int() function to each character in the string 5, effectively converting it into the integer 5. When used with input().split(), it becomes a clean way to read and convert multiple inputs at once:

n1, n2, n3 = map(int, input("Enter 3 numbers separated by space: ").split())
print(f"{n1} + {n2} + {n3} = {n1 + n2 + n3}")  # n1, n2, n3 are integers

Here, each value entered by the user is automatically converted into an integer before being assigned. You can also use map(float, ...) for decimal values or apply custom functions for specific conversions. Since map() returns an iterator, you can convert it into a list using list(map(...)) if you need to store or print all converted values together.

It's also important to note that input() reads one full line at a time. It waits for the user to press Enter before processing the input, even if you type fewer characters. For reading multiple lines, you can use a loop or read from sys.stdin, which is more common in competitive programming or file input scenarios.

  • What is the return type of input()?
    Always a string, regardless of what the user enters.

  • How can you take multiple inputs in a single line?
    By combining input().split() and optionally map() for type conversion.

  • Why is map() preferred over manually converting each value?
    Because it automatically applies conversion to all elements, making code cleaner and faster.

  • What happens if you use + on input values without converting them?
    String concatenation occurs instead of addition.

  • Can you take multiple lines of input using input()?
    Yes, by calling input() repeatedly inside a loop.
